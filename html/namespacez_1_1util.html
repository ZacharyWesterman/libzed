Parse a list of regex symbols into a rule tree.

Parameters
inputThe regex scan symbol list.
parseErrorIf an error is encountered, gives a nonzero value indicating the first error. Otherwise, gives RGX_NO_ERROR (0).
positionThe index of the next scan symbol we are looking at. Should be set to 0 when calling externally.
inTypeThe type of compound rule we are creating. Should be set to 0 when calling externally.
flagsRegex flags for the current level. These are defined in z/util/regex/rgxflag.hpp
Returns
A pointer to an initialized root rule. If not null, deallocation must be handled by the caller.

rgxss z::util::rgxsesc ( uint32_t  ch)
noexcept

Get the appropriate regex symbol from an escape character.

This is called after having read a backslash. So we get results like ‘'s’ → RGX_SPACE,'b' → RGX_BREAK`, etc.

Parameters
chThe character that has been escaped.
Returns
The regex symbol resulting from the escaped character.
Re-Entry:This function is thread-safe. Simultaneous calls will not cause data races.
